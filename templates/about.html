Project Description

The Legal Document Analyzer is an AI-powered tool designed to demystify complex legal documents like Privacy Policies and Terms of Service. It aims to empower users by providing clear, concise summaries and critical insights into these documents.

Our mission is to enhance transparency and understanding for consumers by highlighting essential clauses related to data handling, user rights, and contractual obligations.

How It Works

  1. URL Submission: Users submit a URL to a legal document (e.g., a Privacy Policy) via the main interface.
  2. Web Scraping: The backend Flask application fetches the content of the provided URL, extracts the main text, and cleans it for analysis.
  3. Cache Management: Analyzed documents are cached locally. If a document has been analyzed before and its cached version is up-to-date, the cached result is served immediately, significantly speeding up retrieval.
  4. AI Analysis: The extracted text is then sent to a powerful Gemini large language model. Using a carefully crafted prompt and a structured JSON schema, the AI identifies and extracts key information such as product coverage, data protection measures, user rights, dispute resolution methods, and more.
  5. Citation Extraction: For each piece of extracted information, the AI also attempts to provide a direct quote (citation) from the original document, allowing users to verify the findings.
  6. Results Display: The structured analysis results are then presented on the frontend in an easy-to-read format, including tables and highlighted key points.

How to Use Its APIs

This application exposes several API endpoints for interaction. All responses are in JSON format.

1. Analyze Document

Endpoint: POST /analyze

Description: Initiates the analysis of a legal document from a given URL. If the document is already cached and up-to-date, it returns the result immediately. Otherwise, it starts a background analysis task.

Request Body (JSON):

{
    "url": "https://example.com/privacy-policy"
}

Success Response (Processing):

{
    "job_id": "a1b2c3d4e5...",
    "status": "processing"
}

Success Response (Cached/Completed Immediately):

{
    "job_id": "a1b2c3d4e5...",
    "status": "completed",
    "result": { /* full analysis data */ }
}

Error Response: {"error": "Error message"}

2. Check Job Status

Endpoint: GET /status/<job_id>

Description: Retrieves the current status and progress of an ongoing analysis job using its job_id.

Success Response:

{
    "job_id": "a1b2c3d4e5...",
    "status": "scraping" | "analyzing" | "completed" | "failed",
    "progress": 0-100,
    "error": "Optional error message if status is failed"
}

3. Get Analysis Result

Endpoint: GET /result/<job_id>

Description: Fetches the complete analysis result for a given job_id once the analysis is completed.

Success Response:

{
    "version": "1.0.1.1",
    "url": "https://example.com/privacy-policy",
    "title": "Example Privacy Policy",
    "full_analysis": { /* detailed structured AI analysis */ },
    "document_raw_text": "Full text of the analyzed document...",
    "timestamp": 1678886400.0
}

Error Response (if not completed or failed): {"status": "processing", "message": "Analysis is still in progress."}

4. Get Recent Analyses

Endpoint: GET /recent_analyses

Description: Returns a list of the 5 most recently analyzed documents from the cache, including their URL and title.

Success Response:

[
    {
        "url": "https://example.com/recent-policy-1",
        "title": "Recent Policy One",
        "timestamp": 1678886400.0
    },
    {
        "url": "https://example.com/recent-policy-2",
        "title": "Recent Policy Two",
        "timestamp": 1678886300.0
    }
]

5. Search Cached Documents

Endpoint: GET /search_cached?query=<your_query>

Description: Searches through cached analysis results by matching the query against URLs or titles. Case-insensitive.

Success Response:

[
    {
        "url": "https://example.com/search-result-1",
        "title": "Search Result One",
        "timestamp": 1678886400.0
    },
    {
        "url": "https://example.com/another-matching-doc",
        "title": "Another Document",
        "timestamp": 1678886200.0
    }
]
